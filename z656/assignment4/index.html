Answer to Question 5:

There are 34 "p" elements in the text of this document, however several of these are actually headers of individual books being reviewed. It seems as though all of these headers include the "lb" element (left-bullet?). Searching for "p" elements not including the "lb" element finds 16 results.
<XPATH>count(//body/div/p)</XPATH> or better:
<XPATH>count(//body/div/p[not(child::lb)]</XPATH>

Answer to Question 6:

There are 7 distinct states listed in this document. This xpath needed a little troubleshotting because "Monroe" is erroneousely listed as a state.
<XPATH>count(distinct-values(//region[@type="state"][@reg!="Monroe"])))</XPATH>

Answer to Question 7:

There are 23 distinct cities mentioned in the text. For some reason, the xpath distinct-values(//body//placeName/settlment[@type="city"]) still double counts South Bend and Notre Dame. There does not appear to be any difference in the way these values are spelled within the attribute, and I couldn't figure out a way to elliminate these duplicates using xpath, meaning that to get an acccurate count of cities I had to subtract two from the xpath result.
<XPATH>count(distinct-values(//body//placeName/settlement[@type="city"]))-2</XPATH>

Answer to Question 9:

As with question five, searching for the "lb" element would be one way to count reviews, however I also tried a few other methods. Because each review lists a price for the book under review, counting the number of paragraphs in which the $ symbol appears should reveal how the number of reviews in the issue (16). While  similar xapaths using words needed for title statments could also be used (eg. by searching for paragraphs that contain the words "editor," or "author") those words are more likely to appear within the reviews themselves. The issue with searching for the $ symbol is that it may exclude any books with a price that is TBD (or free books), but it is still the best approximation I can find using xpath alone. 
<XPATH>//body//p[contains(.,"$")]</XPATH>
